# 우슈 모션 평가 앱 — 시스템 설계 (v0.1)

> 목적: AI 모션 인식 기반 우슈 1·2·3단 품세(套路) 평가 SaaS.
> 1차 데모 타깃: 중국 바이어(B2B 평가판).
> 본 문서는 **기초 설계 초안**입니다. 우슈 국가대표 자문자의 피드백을 받기 위한 골격이며,
> "❓전문가 검토 필요" 표시가 붙은 항목은 실제 채점 기준·품세 구성에 대한 의견을 받아 v0.2에서 보강합니다.

---

## 1. 한 줄 가치 제안 (Value Proposition)

**"옆에 국가대표 코치가 앉아 있는 것처럼, 영상을 찍자마자 동작별 점수와 교정 포인트를 받는다."**

- **Who**: 우슈 도장 운영자, 학원 강사, 단증 준비 수련생, (특히) 중국·동남아 우슈 학원 시장.
- **Pain**: 단증 응시 전 객관적인 셀프 평가 어려움, 1:1 코칭 비용·접근성, 강사 한 명이 동시에 여러 명을 정밀 교정하기 어려움.
- **Wedge**: 표준 규정 품세(规定套路)는 동작이 정형화되어 있어 비교 기반 채점이 가능함 → 자유 안무보다 AI에 적합.

## 2. 핵심 평가 도메인 정의

### 2.1 평가 대상 품세 (스코프)

| 단 | 예시 품세 | 영상 길이 | 난이도 |
|----|-----------|-----------|--------|
| 1단 | 초급 장권(初级长拳), 초급 도술(初级刀术) | 1~1.5분 | 기본 보형(步型)·수형(手型) |
| 2단 | 중급 장권, 중급 검술 | 1.5~2분 | 회전·도약 추가 |
| 3단 | 규정 장권(甲组), 규정 남권 | 2~3분 | 난도 동작(腾空·跌扑) 포함 |

> ❓전문가 검토 필요: 한국 우슈협회 단별 규정 품세 목록과 IWUF(国际武联) 평가 기준 중 어느 쪽을 베이스로 할지. MVP는 **1단 1개 품세**부터 시작 권장.

### 2.2 채점 기준 (IWUF 套路 규정 차용)

```
총점 = A (동작 품질) + B (연무 수준) + C (난도)
       ↑ 5점        ↑ 3점        ↑ 2점     = 10점 만점
       감점 방식      가점 방식    가점 방식
```

| 차원 | 측정 방식 | 자동화 난이도 |
|------|-----------|---------------|
| **A. 동작 품질** | 보형 각도, 수형 정확도, 자세 안정성 | ★★☆ (실현 가능) |
| **B. 연무 수준** | 리듬, 박자 일관성, 힘 표현(力点), 정신력(精神) | ★★★★ (어려움) |
| **C. 난도 동작** | 도약 높이·체공시간·회전수, 착지 안정성 | ★★★ (3D 추정 필요) |

**MVP 결정**: A 차원을 1차로 자동화, B/C는 보조 지표만 표시.

### 3.1 왜 이 구조인가

- **온디바이스 포즈 추출**: 영상 원본을 서버로 보내지 않아 (a) 대역폭 절약, (b) 프라이버시, (c) 중국 네트워크 환경 대응. 골격 좌표(JSON)만 업로드.
- **단일 백엔드 + 백오피스 분리**: 자문가가 채점 로직을 직접 손댈 필요 없이 "이 동작은 7점이 아니라 5점"이라는 라벨만 주면 모델/룰이 학습.
- **기준 동작 DB**: 국가대표 시범 영상에서 추출한 포즈 시퀀스가 ground truth. 실시간 비교의 기준점.

## 4. 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 모바일 | **Flutter** | iOS/Android 동시 출시, 중국 안드로이드 분포 대응. 카메라/MediaPipe 플러그인 성숙. |
| 포즈 추정 | **MediaPipe Pose (BlazePose 33 keypoints)** 또는 **MoveNet Lightning** | 모바일 실시간 30~60fps. 무료/오픈소스. |
| 백엔드 | **FastAPI (Python 3.12)** | NumPy/SciPy/scikit-learn 호환, ML 친화. |
| DB | **PostgreSQL** + **TimescaleDB**(포즈 시퀀스용) | 시계열 좌표 저장에 적합. |
| 객체 저장 | **Cloudflare R2** 또는 **S3 호환** | 중국 접근성 고려해 R2 권장. |
| 인증 | **Firebase Auth** + 위챗 로그인(2단계) | 중국 시장 진입 시 위챗 필수. |
| 백오피스 | **Next.js + shadcn/ui** | 자문가가 PC에서 사용. |
| 다국어 | **i18n: ko / zh-CN / en** | 1차 zh-CN 우선. |
| 결제 | **Stripe** + **WeChat Pay** | 평가판은 무료, 본 출시 대비. |

> ❓미결: 3D 포즈가 필요한 동작(도약, 회전수)이 있다면 단일 카메라 2D로는 한계. v1.5에서 다중 카메라 또는 LiDAR(아이폰 Pro) 옵션 검토.

## 5. 데이터 모델 (핵심 테이블)

```sql
-- 단별/품세별 표준 정의
forms (
  id, dan_level (1|2|3), name_ko, name_zh, name_en,
  duration_sec, segments (jsonb)  -- 동작 구간 분할
)

-- 기준(국가대표) 시범 데이터
reference_performances (
  id, form_id, performer_name, video_url,
  pose_sequence (jsonb),  -- [{t: 0.033, joints: {...33 keypoints}}, ...]

  approved_by (FK to experts),
  created_at
)

-- 사용자 응시 기록
attempts (
  id, user_id, form_id, video_url,
  pose_sequence (jsonb),
  scores (jsonb),  -- {A: 4.2, B: 2.1, C: 1.5, total: 7.8, breakdown: [...]}
  feedback (jsonb),
  expert_review_id (nullable, FK)
)

-- 자문가 검토(피드백 루프의 핵심)
expert_reviews (
  id, attempt_id, expert_id,
  ai_score, expert_score,
  delta (jsonb),  -- 동작별 점수 차이
  notes_text,
  used_for_training (boolean)
)

-- 피드백 템플릿 (자문가가 직접 편집)

  id, condition_dsl,  -- 예: "left_knee_angle < 90 AND step_type='gōngbù'"
  message_ko, message_zh, message_en,
  severity (info|warning|critical)
)
```

## 6. 핵심 컴포넌트 상세

### 6.1 채점 엔진 (Scoring Engine)

**입력**: 사용자 포즈 시퀀스 + 기준 포즈 시퀀스 (같은 form_id)
**출력**: 동작별 점수 + 차이 시각화 데이터

알고리즘 파이프라인:
1. **시간 정렬**: DTW(Dynamic Time Warping)로 기준-사용자 시퀀스 정렬. 빠르게/느리게 한 부분 보정.
2. **세그먼트 분할**: forms.segments에 따라 동작 단위로 자른다 (예: "마보충권" 1동, "궁보충권" 2동...).
3. **세그먼트별 평가**:
   - **수형/보형 정확도**: 핵심 관절 각도 차이 (RMSE)
   - **균형**: 무게중심(COM) 흔들림(분산)
   - **범위**: 팔/다리 도달점 좌표 차이
4. **감점 룰 적용**: feedback_templates의 condition_dsl을 평가, 매칭되면 감점 + 메시지.
5. **합산**: A/B/C 점수로 환산.

> ❓전문가 검토 필요: 보형(步型)별 핵심 관절 각도 임계값 — 예) 마보(马步)는 무릎 90도, 발끝 정면. 자문가가 단별 허용 오차를 정의해 줘야 정확.

### 6.2 피드백 생성기 (Feedback Generator)

두 단계 하이브리드:

```
[감점 룰 매칭 결과]
        │
        ▼
[룰 기반 메시지]  ← feedback_templates에서 직접 가져옴
        │
        ▼
[LLM 후처리]      ← Claude/GPT로 자연스러운 한 문단으로 통합
        │
        ▼
"마보 자세에서 왼 무릎이 75도로 너무 굽혀졌습니다.
 90도까지 펴고 발끝을 정면으로 두면 균형이 안정됩니다.
 (해당 구간: 0:23~0:27)"
```

**왜 하이브리드인가**: 순수 LLM은 환각·일관성 문제. 룰만 쓰면 딱딱함. 룰로 사실(어디·무엇·얼마나)을 박고, LLM은 표현만 다듬는다.

### 6.3 백오피스 (자문가 검수 화면)

핵심 화면 3개:
1. **검수 대기열**: AI 채점 결과 중 자문가 확인이 필요한 케이스 우선순위 표시.
2. **나란히 비교 뷰**: 기준 영상 ↔ 사용자 영상 ↔ 골격 오버레이, 슬로 모션, 프레임 단위 이동.
3. **피드백 템플릿 에디터**: 조건(DSL) + 메시지 + 단별 적용 여부 편집.

피드백 루프:
```
사용자 응시 → AI 채점 → (랜덤 샘플링) → 자문가 검수 → 점수/메시지 보정
                                              ↓
                                  delta 데이터로 채점 가중치 학습
                                              ↓
                                          모델 갱신
```

## 7. API 설계 (REST, OpenAPI 3.1 권장)

```
POST   /v1/auth/login
GET    /v1/forms?dan=1                          # 응시 가능한 품세 목록
GET    /v1/forms/{id}/reference                 # 기준 영상 URL + 메타
POST   /v1/attempts                             # 응시 시작 (presigned URL 발급)
POST   /v1/attempts/{id}/pose                   # 포즈 시퀀스 업로드
POST   /v1/attempts/{id}/score                  # 채점 트리거
GET    /v1/attempts/{id}                        # 결과 조회
GET    /v1/attempts/{id}/feedback               # 다국어 피드백

# 백오피스
GET    /v1/admin/reviews/pending
POST   /v1/admin/reviews/{id}                   # 자문가 채점 제출
PUT    /v1/admin/feedback-templates/{id}
POST   /v1/admin/forms/{id}/reference           # 기준 영상 등록
```

## 8. UX 흐름 (사용자 시나리오)

```
1. 단/품세 선택  →  2. 시범 영상 시청  →  3. 따라하기/촬영
                                                  │
                                                  ▼
4. 결과 화면 ← 5. 채점 (5~10초)
   ├─ 총점 + A/B/C
   ├─ 동작별 타임라인 (빨강=감점)
   ├─ 골격 비교 슬로모션
   └─ 피드백 카드 3~5개 (가장 큰 감점부터)
                                                  │
                                                  ▼
6. 재응시 / 자문가 정밀 검수 신청 (유료 옵션)
```

**중요한 UX 원칙**:
- **첫 화면 5초 내에 점수**: 신뢰의 근거.
- **"왜 깎였는지"가 보여야 함**: 골격 오버레이 차이가 핵심.
- **기준 시범 영상은 항상 옆에**: "내 동작이 저렇게 되어야 하는구나" 학습 효과.

## 9. MVP 범위와 단계별 로드맵

### Phase 0 — 기반 (2주)
- [ ] 1단 1개 품세(초급 장권) 선정
- [ ] 국가대표 자문가의 시범 영상 1편 촬영 (정면·측면 2앵글)
- [ ] 영상에서 포즈 시퀀스 추출 → 기준 데이터 생성
- [ ] 자문가와 함께 동작 구간 분할 + 핵심 채점 포인트 5~10개 정의

### Phase 1 — 기술 검증 (3주)
- [ ] Flutter 앱 골격 + MediaPipe 포즈 추출 PoC
- [ ] FastAPI 백엔드 + DTW 채점 엔진 PoC
- [ ] 자문가가 5명의 다양한 수련생을 직접 채점 → AI 결과와 상관계수 측정 (목표: r ≥ 0.7)

### Phase 2 — 자문가 피드백 루프 (3주)
- [ ] 백오피스 검수 화면
- [ ] 피드백 템플릿 에디터
- [ ] 자문가가 50건 이상 검수 → 룰/임계값 보정
- [ ] zh-CN 번역 1차 완료

### Phase 3 — 중국 바이어 평가판 (2주)
- [ ] 위챗 로그인
- [ ] 중국 CDN/리전 배포 (R2 + 대체 도메인)
- [ ] 데모 계정 + 안내 자료 (zh-CN)
- [ ] 사용 로그 분석 대시보드

### Phase 4 (선택) — 확장
- 2단·3단 품세 추가
- 난도 동작(도약/회전) 자동 평가
- 그룹/도장 관리 기능

## 10. 자문가(국가대표) 피드백을 어떻게 코드에 반영하는가

이 시스템의 차별점은 **"전문가 피드백을 데이터로 받아서 자동 반영"**입니다.

| 자문가가 주는 피드백 형태 | 시스템이 받는 방식 | 반영 방법 |
|---|---|---|
| "마보 무릎 각도는 85~95도가 정상" | 백오피스에서 임계값 입력 | feedback_templates 자동 업데이트 |
| "이 응시는 7점이 아니라 5점" | 백오피스에서 점수 보정 | expert_reviews에 delta 저장 → 채점 가중치 재학습 |
| "이 메시지는 어조가 너무 강함" | 메시지 템플릿 직접 편집 | 즉시 반영 |
| "이 동작은 자동 평가 불가" | 평가 제외 토글 | 해당 segment를 manual_only로 표시 |

→ 자문가는 코드를 몰라도 됩니다. **"규칙 + 점수 + 메시지"** 세 가지만 백오피스로 입력하면 시스템이 학습.

## 11. 위험 요소 & 미해결 질문

### 기술 리스크
- **단일 카메라 2D 포즈의 한계**: 회전, 깊이, 도약 높이 측정 부정확 → MVP는 정면 촬영만 권장하고 명시.
- **다양한 체형/촬영 거리**: 정규화 필요 (어깨 너비 기준 스케일링).
- **조명/배경**: MediaPipe는 비교적 강건하나, 어두운 도장에서 정확도 저하 가능 → 촬영 가이드 UX 필수.

### 비즈니스 리스크
- **중국 데이터 규제**: 사용자 영상이 중국 외부로 나가면 안 될 수 있음 → 중국 리전 분리 필요할 수 있음. 변호사 자문 권장.
- **위챗 결제 가맹점 등록**: 한국 법인으로 가능 여부 확인.
- **저작권**: 국가대표 시범 영상의 사용권/초상권 계약서 필요.

### ❓전문가 검토 필요 항목
1. 1단 MVP 품세는 무엇으로 시작할까요? (초급 장권 / 초급 도술 / 그 외)
2. 동작별 채점 가중치를 어떻게 두는 게 현장 단증 채점과 가까운가요?
3. 자동화가 절대 불가능한 채점 항목은 무엇인가요? (정신·기세 등) — 솔직히 표시하는 게 신뢰에 도움.
4. 중국 우슈와 한국 우슈의 채점 기준 차이가 있나요? (있다면 form 단위로 분기)
5. 평가판을 보여줄 중국 바이어의 기대 수준 — 단순 스코어? 정밀 코칭? 학원 운영 도구?
